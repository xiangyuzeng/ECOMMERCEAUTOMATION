## 准备工作

### 演示前必须确认

| 项目 | 检查方式 | 状态 |
|------|----------|------|
| Docker Desktop 已安装并运行 | 查看任务栏/菜单栏图标 | ☐ |
| AdsPower 已安装并运行 | 查看任务栏/菜单栏图标 | ☐ |
| AdsPower 中已登录 SellerSprite 账号 | 打开 AdsPower 浏览器验证 | ☐ |
| `.env` 文件已配置 API Key | 用文本编辑器打开 `.env` 检查 | ☐ |
| 网络连接正常 | 能访问 amazon.com | ☐ |

### AdsPower 配置确认

在演示前，请确认以下三个值已正确填入 `.env` 文件：

```env
ADSPOWER_API_URL=http://localhost:50325
ADSPOWER_API_KEY=你的API密钥
ADSPOWER_PROFILE_ID=你的配置文件ID
```

**获取方法**：
1. **API URL**：默认为 `http://localhost:50325`，无需修改
2. **API Key**：打开 AdsPower → 右上角齿轮图标 → API → 复制 API Key
3. **Profile ID**：在 AdsPower 主界面 → 浏览器配置列表 → 找到你用来登录卖家精灵的配置 → 复制其 ID（类似 `k1amt6rq`）

### 准备一个演示用的产品链接

选择一个 Amazon US 产品链接用于演示，建议选择：
- 有一定评论数的产品（>10 条评论）
- 非敏感类目（避免成人用品、药品等）
- 中等竞争度类目（采集速度更快）

**推荐演示用链接**：
```
https://www.amazon.com/dp/B094PZTFMB
```
（COLD POSH 男士真丝内衣 — 系统内置品牌产品）

## 第一步：启动系统

### Mac 操作

打开终端（Terminal），输入以下命令：


### 分类规则说明

**一级分类**（按搜索量）：
- **大词**：月搜索量 > 50,000
- **核心关键词**：10,000 ~ 50,000
- **功能/场景/人群词**：3,000 ~ 10,000
- **长尾词**：< 3,000 或词数 >= 4 个
- **竞品品牌词**：含竞品品牌名

**二级分类**（按转化表现）：
- **主力出单词**：购买率 > 2% 且自然排名 <= 20
- **潜力测试词**：月搜索量 > 5,000 且排名 > 50
- **流量/防御词**：月搜索量 > 10,000 但购买率 < 0.5%
- **无效/亏损词**：无购买、无排名、CPC > $2

## 第九步：查看关键词 Gap 分析

点击标签页 **"关键词差距分析 Keyword Gap"**。

### 你会看到

与主要竞品的关键词覆盖差距对比：

| 关键词 | 我的排名 | 竞品排名 | 差距类型 | 月搜索量 | 优先级 | 建议操作 |
|--------|----------|----------|----------|----------|--------|----------|
| silk pajamas | — | 5 | MISSING | 45,000 | 高 | 立即投放广告+埋词标题 |
| men's loungewear | 85 | 12 | CATCHUP | 22,000 | 中 | 提升广告预算，优化排名 |
| luxury silk | 3 | 28 | DEFEND | 8,500 | 低 | 保持现有排名，防御竞品 |

### 三种差距类型

- **MISSING（缺失）**：竞品有排名但我们没有 — 最大机会！
- **CATCHUP（追赶）**：我们排名落后于竞品 — 需要提升
- **DEFEND（防御）**：我们排名领先竞品 — 需要保护

优先级分数 = 月搜索量 x 购买率 x 权重系数

> **演示话术**："Gap 分析是最有价值的功能之一 — 它直接告诉你竞品有哪些流量来源是你还没有覆盖的，以及应该优先投入哪些词。"

## 演示常见问题处理

### 问题 1：页面打不开（localhost:3000 无响应）

```bash
# 检查容器是否在运行
docker compose ps

# 如果没有运行，重新启动
docker compose up -d

# 查看错误日志
docker compose logs --tail 20
```

### 问题 2：采集过程中浏览器崩溃

**现象**：进度条停止更新，某个任务一直显示"运行中"

**处理方式**：
1. 系统会自动检测浏览器状态并尝试恢复
2. 如果 60 秒内没有恢复，系统会跳过该任务继续下一个
3. 已采集的数据不会丢失
4. 采集完成后仍会生成报表（缺失的数据标记为 N/A）

### 问题 3：AdsPower 连接失败

**现象**：点击"开始采集"后提示 AdsPower 无法连接

**处理方式**：
1. 确认 AdsPower 程序已打开并运行
2. 在 AdsPower 中检查 API 端口是否为 50325
3. 检查 `.env` 文件中的 API Key 是否正确
4. 重启 AdsPower 后重试

### 问题 4：采集的数据量为 0

**可能原因**：
- SellerSprite 账号未登录或已过期
- 产品 ASIN 在 SellerSprite 中无数据
- 网络连接中断

**处理方式**：
1. 打开 AdsPower 浏览器，手动访问 SellerSprite 确认已登录
2. 尝试换一个有更多数据的产品
3. 使用"手动上传"功能补充数据

## 附录：系统 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/discover` | POST | 启动产品发现（传入 Amazon URL） |
| `/api/discover` | GET | 查询发现进度 |
| `/api/collect` | POST | 启动自动采集 |
| `/api/collect` | GET | 查询采集进度（2.5 秒轮询） |
| `/api/process` | POST | 手动触发数据处理管道 |
| `/api/status` | GET | 获取系统状态（文件数、上次运行时间） |
| `/api/data` | GET | 加载仪表盘数据（6 个 JSON） |
| `/api/products` | GET | 列出所有已分析的产品 |
| `/api/products` | POST | 切换活跃产品 |
| `/api/upload` | POST | 手动上传数据文件 |
| `/api/config` | GET | 获取当前产品配置 |
| `/api/profiles` | GET | 列出可用的 Chrome 配置 |
